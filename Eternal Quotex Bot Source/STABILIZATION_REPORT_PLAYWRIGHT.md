# Technical Audit & Stabilization Report: Quotex Connection Layer

## Overview
This report documents the architectural overhaul of the **Eternal Quotex Bot**'s connection layer, completed on April 29, 2026. The primary goal was to resolve persistent login stalls, eliminate external dependencies, and ensure a zero-fail initialization process.

## 1. Dual-Mode Connection Engine
The backend (`live.py`) has been refactored to use a tiered connection strategy:

1.  **Tier 1: Selenium (undetected-chromedriver)**
    *   **Primary Engine**: Remains the default due to its ability to bypass advanced bot detection.
    *   **Improvements**: Replaced the brittle `quotexpy` internal login hook (which used `document.getElementsByName("email")[1].value`) with a dynamic JS injection layer.
    *   **Bug Fix**: Resolved a critical `SyntaxError` where `const pass = ...` was used (reserved word).

2.  **Tier 2: Playwright Bridge (Fallback)**
    *   **Auto-Fallback**: Triggered automatically if Selenium fails to launch, times out, or encounters a selector mismatch.
    *   **Zero-Dependency**: Configured to use the system's existing browser (Chrome/Edge) via the `channel` parameter. This avoids the 200MB+ overhead of bundled Chromium binaries.
    *   **Websocket Interception**: The bridge intercepts market data frames directly if the primary websocket client fails to establish a handshake.

## 2. Playwright Bridge Implementation (`playwright_bridge.py`)
A standalone bridge module was created to provide a lightweight, portable alternative to Selenium.

*   **Logic**:
    *   Detects installed browsers (Chrome, Chrome Stable, Edge).
    *   Uses `playwright.async_api` with `channel="chrome"` or `channel="msedge"`.
    *   Implements a multi-layered selector discovery for login fields (Email, Password, PIN).
    *   Extracts the `ssid` (token) and cookies directly from the browser context.
*   **Integration**:
    *   The `LiveQuotexBackend` in `live.py` uses this bridge to obtain authentication credentials.
    *   These credentials are then "hot-loaded" into the `quotexpy` client, skipping the brittle internal login flow.

## 3. Build System Optimization (`Eternal Quotex Bot.spec`)
The PyInstaller specification was updated to ensure the Playwright driver is functional within the `.exe` without including unnecessary browser binaries.

*   **Payload**: Includes `playwright/driver` files only.
*   **Size**: Reduced the potential executable size from ~250MB+ to ~26MB.
*   **Portability**: The resulting `.exe` is a single file that runs on any Windows machine with Chrome or Edge installed.

## 5. Guidance for Future AI Assistants
*   **Connection Issues**: If the bot fails to connect, check `.quotexpy.log`. It will indicate if it switched to Playwright.
*   **Selector Updates**: If Quotex changes their DOM, update the selector arrays in BOTH `live.py` (Selenium path) and `playwright_bridge.py` (Playwright path).
*   **Headless Mode**: Quotex aggressively detects headless browsers. The "Headless" setting in the UI actually launches a minimized visible window (`--start-minimized`) to remain undetected.
